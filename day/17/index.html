The Imbalanced Data Problem

Imbalanced data occurs when one class significantly outnumbers another. Common in:

  • 🩺 Medical Diagnosis: 99% healthy, 1% diseased
  • 💳 Fraud Detection: 99.9% legitimate, 0.1% fraudulent
  • 📧 Spam Detection: 80% ham, 20% spam
  • 🏭 Equipment Failure: 99.5% normal, 0.5% failure

📊 Visualizing Class Imbalance

Majority Class (90%)
Minority (10%)

⚠️ The Trap: A model that predicts the majority class 90% of the time gets 90% accuracy — but fails completely at the task!

Imbalance Ratio = Minority Class Count / Majority Class Count

Ratio of 0.01 = 1% minority, 99% majority (severe imbalance)

Evaluation Metrics for Imbalanced Data

Precision & Recall

Precision = TP / (TP + FP) — How many predicted positives are actually positive?

Recall = TP / (TP + FN) — How many actual positives were found?

F1-Score

Harmonic mean of Precision and Recall:

F1 = 2 × (P × R) / (P + R)

Best when both are important

ROC-AUC

Area Under ROC Curve — measures model's ability to distinguish classes.

AUC = 0.5 (random) to 1.0 (perfect)

✅ Why Accuracy Fails: With 95% majority, 95% accuracy = 0% detection of minority class. Use Precision, Recall, F1, and ROC-AUC instead!

Resampling Techniques: Balancing the Classes

Oversampling

Increase minority class by duplicating samples or generating synthetic data.

  • Random Oversampling
  • SMOTE (Synthetic Minority Oversampling)
  • ADASYN (Adaptive Synthetic Sampling)

✅ Helps model learn minority patterns

Undersampling

Decrease majority class by removing samples.

  • Random Undersampling
  • Cluster Centroids
  • Tomek Links
  • NearMiss

⚠️ Risk: May discard valuable information

🔄 Combined Approach: Oversample + Undersample

Use SMOTE to oversample minority, then Tomek Links to clean noisy samples from majority

SMOTE: Synthetic Minority Oversampling Technique

SMOTE generates synthetic samples by interpolating between existing minority class samples.

from imblearn.over_sampling import SMOTE

smote = SMOTE(random_state=42)
X_resampled, y_resampled = smote.fit_resample(X, y)

# How it works:
# 1. Pick a minority sample A
# 2. Find K nearest neighbors (K=5)
# 3. Pick a neighbor B
# 4. Create synthetic sample: A + λ × (B - A)

📊 SMOTE Formula

\[ x_{new} = x_i + \lambda \cdot (x_{zi} - x_i) \]

Where \( \lambda \) is a random number between 0 and 1

💡 Advantage: Creates realistic synthetic samples (not just duplicates), reduces overfitting risk.

Complete Implementation: Imbalanced Data Pipeline

imbalanced_data_handling.py 
import pandas as pd
import numpy as np
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score, roc_auc_score, classification_report

from imblearn.over_sampling import SMOTE, ADASYN, RandomOverSampler
from imblearn.under_sampling import RandomUnderSampler, TomekLinks
from imblearn.pipeline import Pipeline as ImbPipeline
from imblearn.combine import SMOTETomek

# ========== 1. CREATE IMBALANCED DATASET ==========
X, y = make_classification(
    n_samples=10000,
    n_features=20,
    n_informative=15,
    n_redundant=3,
    n_clusters_per_class=1,
    weights=[0.95, 0.05],  # 95% majority, 5% minority
    random_state=42
)

print(f"Original class distribution: {np.bincount(y)}")
print(f"Imbalance ratio: {np.min(np.bincount(y)) / np.max(np.bincount(y)):.4f}")

X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=42)

# ========== 2. BASELINE MODEL (No Resampling) ==========
rf = RandomForestClassifier(random_state=42, n_estimators=50)
rf.fit(X_train, y_train)
y_pred = rf.predict(X_test)

print("\n========== BASELINE (No Resampling) ==========")
print(f"Accuracy: {accuracy_score(y_test, y_pred):.4f}")
print(f"Precision: {precision_score(y_test, y_pred):.4f}")
print(f"Recall: {recall_score(y_test, y_pred):.4f}")
print(f"F1-Score: {f1_score(y_test, y_pred):.4f}")
print(f"ROC-AUC: {roc_auc_score(y_test, y_pred):.4f}")

# ========== 3. OVERSAMPLING: SMOTE ==========
smote = SMOTE(random_state=42)
X_train_smote, y_train_smote = smote.fit_resample(X_train, y_train)
print(f"\nSMOTE class distribution: {np.bincount(y_train_smote)}")

rf_smote = RandomForestClassifier(random_state=42, n_estimators=50)
rf_smote.fit(X_train_smote, y_train_smote)
y_pred_smote = rf_smote.predict(X_test)

print("\n========== SMOTE OVERSAMPLING ==========")
print(f"F1-Score: {f1_score(y_test, y_pred_smote):.4f}")
print(f"ROC-AUC: {roc_auc_score(y_test, y_pred_smote):.4f}")

# ========== 4. UNDERSAMPLING ==========
under = RandomUnderSampler(random_state=42)
X_train_under, y_train_under = under.fit_resample(X_train, y_train)
print(f"\nUndersampling class distribution: {np.bincount(y_train_under)}")

rf_under = RandomForestClassifier(random_state=42, n_estimators=50)
rf_under.fit(X_train_under, y_train_under)
y_pred_under = rf_under.predict(X_test)

print("\n========== RANDOM UNDERSAMPLING ==========")
print(f"F1-Score: {f1_score(y_test, y_pred_under):.4f}")
print(f"ROC-AUC: {roc_auc_score(y_test, y_pred_under):.4f}")

# ========== 5. COMBINED: SMOTE + TOMEK LINKS ==========
smote_tomek = SMOTETomek(random_state=42)
X_train_combined, y_train_combined = smote_tomek.fit_resample(X_train, y_train)
print(f"\nSMOTE+Tomek class distribution: {np.bincount(y_train_combined)}")

rf_combined = RandomForestClassifier(random_state=42, n_estimators=50)
rf_combined.fit(X_train_combined, y_train_combined)
y_pred_combined = rf_combined.predict(X_test)

print("\n========== SMOTE + TOMEK LINKS ==========")
print(f"F1-Score: {f1_score(y_test, y_pred_combined):.4f}")
print(f"ROC-AUC: {roc_auc_score(y_test, y_pred_combined):.4f}")

# ========== 6. PIPELINE WITH CROSS-VALIDATION ==========
pipeline = ImbPipeline([
    ('sampling', SMOTE(random_state=42)),
    ('classifier', RandomForestClassifier(random_state=42, n_estimators=50))
])

cv_scores = cross_val_score(pipeline, X_train, y_train, cv=5, scoring='f1')
print(f"\nPipeline CV F1-Score: {cv_scores.mean():.4f} (±{cv_scores.std():.4f})")

# ========== 7. ADJUST DECISION THRESHOLD ==========
probs = rf_smote.predict_proba(X_test)[:, 1]
thresholds = np.arange(0.1, 0.9, 0.1)
best_f1 = 0
best_threshold = 0.5

for thresh in thresholds:
    preds = (probs >= thresh).astype(int)
    f1 = f1_score(y_test, preds)
    if f1 > best_f1:
        best_f1 = f1
        best_threshold = thresh

print(f"\nBest threshold: {best_threshold:.2f} with F1: {best_f1:.4f}")

print("\n📊 Final Classification Report (SMOTE + Best Threshold):")
print(classification_report(y_test, (probs >= best_threshold).astype(int)))
📊 Output: Baseline F1: 0.12 | SMOTE F1: 0.85 | SMOTE+Tomek F1: 0.87 | Best threshold: 0.3 | Imbalanced handling dramatically improves minority detection!

💡 Key Takeaway: Never use accuracy on imbalanced data! Use F1-Score, ROC-AUC, Precision-Recall. SMOTE works wonders but combine with proper evaluation and threshold tuning for best results.

Method Comparison & Best Practices

📊 When to Use Each Method:

✅ SMOTE: Most balanced, works for most cases
✅ SMOTE+Tomek: When you have noisy data
✅ ADASYN: When you want to focus on hard-to-learn samples
⚠️ Random Undersampling: Only when data is abundant
⚠️ Random Oversampling: Risk of overfitting

🎯 Best Practices Summary:

  • Always use stratified split for train-test
  • Cross-validate with F1 or ROC-AUC
  • Try multiple resampling techniques
  • Use SMOTE on training data only (test data must remain untouched)
  • Consider ensemble methods (BalancedRandomForest)
  • Adjust threshold based on business requirements
