History of Pandas: From Finance to Data Science

Who Created Pandas?

  • 👤 Wes McKinney - Created Pandas in 2008 while working at AQR Capital Management
  • 🏢 Needed a tool for financial data analysis (time series, cross-sectional data)
  • 📅 First open-source release: January 2008
  • ⭐ Name "Pandas" derived from "Panel Data" (econometrics term)

Why Was Pandas Created?

  • ◆ Excel and SQL were insufficient for complex data workflows
  • ◆ R had great data frames, Python lacked similar tools
  • ◆ Need for labeled data structures (not just NumPy arrays)
  • ◆ Handle missing data gracefully
  • ◆ Built on top of NumPy for performance

📅 Pandas Evolution Timeline

2008: Wes McKinney starts Pandas at AQR
2009: First public release (v0.1)
2012: v0.8.0 - Major improvements, DataFrame becomes core
2020: v1.0.0 - Stable API, production-ready
Today: 30M+ downloads/month, essential for data science

Series: One-Dimensional Labeled Array

A Series is like a NumPy array with labels (index). It can hold any data type and handles missing values.

import pandas as pd

s = pd.Series([10, 20, 30, 40], index=['a', 'b', 'c', 'd'])
print(s)
# a    10
# b    20
# c    30
# d    40
# dtype: int64

Series Attributes:

s.values→ array([10,20,30,40])
s.index→ Index(['a','b','c','d'])
s.dtype→ int64
s['b']→ 20 (label-based access)

DataFrame: Two-Dimensional Labeled Data Structure

A DataFrame is a 2D labeled data structure with columns of potentially different types. Think of it as a spreadsheet or SQL table.

NameAgeCitySalary
0Alice25New York75000
1Bob30Boston82000
2Charlie35Chicago95000
3Diana28Seattle88000

Creating a DataFrame:

df = pd.DataFrame({
    'Name': ['Alice', 'Bob', 'Charlie'],
    'Age': [25, 30, 35],
    'City': ['NYC', 'BOS', 'CHI']
})

Key Attributes:

df.shape → (rows, columns)
df.columns → column names
df.index → row labels
df.info() → data types & memory

Essential DataFrame Operations

🔍 Viewing Data

df.head(n) - First n rows
df.tail(n) - Last n rows
df.sample(n) - Random n rows
df.describe() - Statistical summary

🎯 Selecting Data

df['column'] - Single column (Series)
df[['col1','col2']] - Multiple columns
df.loc['label'] - Label-based indexing
df.iloc[position] - Position-based indexing
df[df['age'] > 30] - Boolean filtering

🧹 Cleaning Data

df.isnull().sum() - Count missing values
df.dropna() - Remove missing values
df.fillna(value) - Fill missing values
df.drop_duplicates() - Remove duplicates
df.rename(columns={'old':'new'})

📊 Grouping & Aggregating

df.groupby('column').mean()
df.groupby(['col1','col2']).agg({'col3':'sum'})
df.pivot_table(values='col', index='row', columns='col')

🔗 Merging Data

pd.concat([df1, df2]) - Stack vertically
pd.merge(df1, df2, on='key') - SQL-like join
df1.join(df2, how='inner')

📈 Apply Functions

df['new'] = df['col'].apply(lambda x: x*2)
df.apply(np.mean, axis=0) - Column-wise
df.applymap(func) - Element-wise

Pandas in Action: Complete Demo

pandas_demo.py 
import pandas as pd
import numpy as np

# ========== CREATE DATAFRAME ==========
df = pd.DataFrame({
    'Name': ['Alice', 'Bob', 'Charlie', 'Diana', 'Eve'],
    'Age': [25, 30, 35, 28, 32],
    'Salary': [70000, 85000, 95000, 80000, 90000],
    'Department': ['Sales', 'Engineering', 'Engineering', 'Sales', 'HR']
})

print("First 3 rows:")
print(df.head(3))

# ========== BASIC INFO ==========
print(f"\nShape: {df.shape}")
print(f"Columns: {list(df.columns)}")

# ========== SELECTION & FILTERING ==========
engineers = df[df['Department'] == 'Engineering']
print(f"\nEngineering dept:\n{engineers}")

# ========== GROUP BY AGGREGATION ==========
dept_stats = df.groupby('Department')['Salary'].agg(['mean', 'min', 'max'])
print(f"\nDepartment Salary Stats:\n{dept_stats}")

# ========== ADD NEW COLUMN ==========
df['Bonus'] = df['Salary'] * 0.10
print(f"\nWith Bonus:\n{df[['Name', 'Salary', 'Bonus']]}")

# ========== STATISTICAL SUMMARY ==========
print(f"\nStatistical Summary:\n{df.describe()}")
📊 Output: DataFrame with 5 rows, 4 columns | GroupBy shows dept averages | Pandas makes data analysis effortless!

💡 Pandas vs Excel vs SQL: Pandas combines the best of spreadsheets (interactive) and databases (powerful queries) with Python's flexibility. It's the industry standard for data manipulation!

Pandas: Strengths & Limitations

✅ Advantages

  • Intuitive data structures (Series, DataFrame)
  • Handles missing data beautifully
  • Fast I/O for many formats (CSV, Excel, SQL, JSON, Parquet)
  • Powerful time series functionality
  • Rich ecosystem of plotting, stats, and ML integration
  • Seamless integration with NumPy, Matplotlib, scikit-learn

❌ Limitations

  • Memory hungry for very large datasets (>10GB)
  • Slower than specialized tools like Polars for some operations
  • Chained indexing can produce warnings
  • Learning curve for complex operations
  • Not designed for distributed computing (use Dask/Spark)

🔗 Libraries Built on/with Pandas

seaborn - Statistical viz
scikit-learn - ML pipelines
plotly - Interactive viz
statsmodels - Statistics
geopandas - Spatial data
Dask - Distributed pandas
