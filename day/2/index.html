Why not Linear Regression for Classification?

Linear Regression Limitations

  • Predicts values beyond [0,1] → not meaningful for probabilities.
  • Sensitive to outliers, shifts decision boundary incorrectly.
  • Assumes linear relationship — classification boundaries are rarely linear in probability space.
❌ For binary classification (0/1), linear regression can give predictions like -0.3 or 1.5.

Logistic Regression Advantage

  • Outputs probability between 0 and 1 using Sigmoid Function.
  • Robust to outliers with proper thresholding.
  • Decision boundary based on log-odds, natural fit for classification.
✅ Probability P(y=1|x) = 1 / (1 + e^{-z}) where z = θᵀx

The Sigmoid (Logistic) Function

\[ \sigma(z) = \frac{1}{1 + e^{-z}} \]

where z = w₁x₁ + w₂x₂ + ... + b

The sigmoid function squashes any real-valued number into the range (0, 1), which can be interpreted as probability. When z=0 → σ(z)=0.5 → decision boundary.

z → +∞ ⇒ σ→1
z → -∞ ⇒ σ→0

Decision Boundary & Log-Odds

Linear Decision Boundary

Logistic regression assumes a linear boundary in feature space: \( w_1x_1 + w_2x_2 + b = 0 \)

Logit function:
\[ \ln\left(\frac{p}{1-p}\right) = w_0 + w_1x_1 + w_2x_2 \]

If log-odds > 0 → class 1 (probability > 0.5)

▶ Prediction rule: If P(y=1|x) ≥ 0.5 → Class 1, else Class 0.

Hands-on: Logistic Regression with Scikit-learn

logistic_tutorial.py 
from sklearn.linear_model import LogisticRegression
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score, classification_report

# Load dataset
data = load_breast_cancer()
X_train, X_test, y_train, y_test = train_test_split(data.data, data.target, random_state=42)

# Train logistic regression
model = LogisticRegression(max_iter=1000)
model.fit(X_train, y_train)

# Predict probabilities
y_pred = model.predict(X_test)
probs = model.predict_proba(X_test)[:, 1]

print(f"Accuracy: {accuracy_score(y_test, y_pred):.3f}")
print(classification_report(y_test, y_pred, target_names=data.target_names))
🧠 Output → Accuracy: 0.973, Precision & Recall > 0.96

💡 Key takeaway: Logistic Regression estimates probabilities, but final classification depends on chosen threshold (default = 0.5). You can adjust threshold to favor precision vs recall.

# 📌 Day 2: Logistic Regression Deep Dive

  • ✅ Logistic regression predicts probabilities using the sigmoid function.
  • ✅ Decision boundary is linear in feature space; probabilities mapped via log-odds.
  • ✅ Metrics to evaluate classification: Confusion Matrix, Precision, Recall, F1, ROC-AUC.
  • ✅ Unlike linear regression, outputs are bounded [0,1] → ideal for binary classification.
  • ✅ Can be extended to multiclass using One-vs-Rest (OvR) strategy.

✦ Real-world: spam detection, medical diagnosis, credit default.
