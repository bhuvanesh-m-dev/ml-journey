Bias vs Variance: The Core Tradeoff

High Bias

Error from wrong assumptions. Model is too simple, misses relevant patterns.

Characteristics:

  • Underfits the training data
  • High training error & high test error
  • Too rigid / linear for complex data
📌 Example: Linear model on non-linear data

High Variance

Sensitivity to small fluctuations. Model learns noise instead of signal.

Characteristics:

  • Overfits the training data
  • Low training error, high test error
  • Too complex, wiggly decision boundaries
📌 Example: Deep decision tree on small dataset

\[ \text{Total Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Error} \]

The fundamental decomposition of prediction error

Underfitting: The High Bias Problem

🔴 Symptoms of Underfitting

  • Poor performance on training data
  • Poor performance on test/validation data
  • Model is too simple for underlying pattern

📈 How to fix:

  • Increase model complexity
  • Add more features / polynomial features
  • Decrease regularization
  • Train longer (for iterative algorithms)

Overfitting: The High Variance Problem

🟣 Symptoms of Overfitting

  • Excellent training performance
  • Poor test/validation performance
  • Model memorizes noise & outliers

🛠️ How to fix:

  • Get more training data
  • Reduce model complexity
  • Apply regularization (L1/L2)
  • Early stopping / pruning
  • Cross-validation & ensemble methods

The Bias-Variance Tradeoff Curve

As model complexity increases, bias decreases (model fits better) but variance increases (model becomes more sensitive). The goal is to find the sweet spot where total error is minimized.

How to Diagnose & Fix

Underfitting (High Bias)

→ Increase model complexity (more layers, features)
→ Add polynomial features (interaction terms)
→ Decrease regularization strength
→ Train for more iterations / epochs
→ Remove feature selection constraints

Overfitting (High Variance)

→ Increase training data (or use data augmentation)
→ Reduce model complexity (simpler architecture)
→ Apply regularization (L1 Lasso, L2 Ridge)
→ Early stopping / cross-validation
→ Ensemble methods (Bagging, Random Forest)
→ Feature selection / dimensionality reduction

🔗 Regularization: The Balancing Act

Regularization techniques (L1 Lasso, L2 Ridge) add a penalty term to the loss function, effectively reducing variance at the cost of slightly increasing bias. This helps move from overfitting toward the optimal tradeoff point.

# ⚖️ Day 5: Bias-Variance Tradeoff — Core Takeaways

  • ✅ Bias: Error from wrong assumptions. High Bias → Underfitting
  • ✅ Variance: Sensitivity to training data. High Variance → Overfitting
  • ✅ Total Error = Bias² + Variance + Irreducible Error
  • ✅ Sweet spot: Balance between underfitting and overfitting via cross-validation
  • ✅ Regularization reduces variance (increases bias slightly) to combat overfitting
  • ✅ More data reduces variance, doesn't affect bias much

📌 Real-world impact: Model selection, hyperparameter tuning, generalization to unseen data.
