Support Vector Machines (SVM)

Loading maximum margin classifier...

Key Concepts: Hyperplane · Support Vectors · Margin

Hyperplane

Decision boundary that separates classes. In 2D, it's a line; in 3D, a plane; in higher dimensions, a hyperplane.

w·x + b = 0

Support Vectors

Data points closest to the hyperplane. They "support" the decision boundary — only these matter for defining the margin.

Points where w·x + b = ±1

Maximum Margin

The distance between the hyperplane and the nearest support vectors. SVM maximizes this margin for best generalization.

Margin = 2/||w||

Linear SVM: Maximizing the Margin

\[ \min_{w,b} \frac{1}{2}||w||^2 \quad \text{s.t.} \quad y_i(w \cdot x_i + b) \geq 1 \]

Optimization problem: maximize margin (minimize ||w||) with constraints

  • ✓ Assumes data is linearly separable
  • ✓ Finds the hyperplane with maximum distance to nearest points
  • ✓ Only support vectors influence the boundary

🔧 Soft Margin SVM (Non-separable data)

When data isn't perfectly separable, SVM uses soft margin with a C hyperparameter:

Small C:Wider margin, more tolerance for misclassifications (lower variance)
Large C:Narrow margin, fewer misclassifications (may overfit)

Non-Linear SVM: The Kernel Trick

🔄 The Problem: Linear Separability

Real-world data is often not linearly separable. XOR problem, concentric circles, etc., cannot be separated by a straight line.

Linear classifier fails on XOR:

(0,0) → 0
(1,1) → 0
(0,1) → 1
(1,0) → 1

✨ Solution: Kernel Trick

Maps data to a higher-dimensional space where it becomes linearly separable, without explicitly computing the transformation.

\[ K(x_i, x_j) = \phi(x_i) \cdot \phi(x_j) \]

Compute dot product in high-dim space efficiently

3D Space: Mapping to Higher Dimensions

Popular Kernel Functions

Linear: K(x, x') = x·x'
Polynomial: K(x, x') = (x·x' + c)^d
RBF (Radial Basis): K(x, x') = exp(-γ||x - x'||²)
Sigmoid: K(x, x') = tanh(α x·x' + c)

💡 The Kernel Trick: Allows SVM to find non-linear decision boundaries without computationally expensive transformations. RBF kernel can create infinitely dimensional feature spaces!

Implementation: SVM with Different Kernels

svm_kernel_demo.py 
from sklearn.svm import SVC
from sklearn.datasets import make_moons, load_breast_cancer
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import accuracy_score

# Generate non-linear dataset (moons)
X, y = make_moons(n_samples=300, noise=0.1, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=42)

# Scale features (IMPORTANT for SVM)
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

# Linear Kernel
svm_linear = SVC(kernel='linear', C=1.0)
svm_linear.fit(X_train_scaled, y_train)
print(f"Linear SVM Accuracy: {accuracy_score(y_test, svm_linear.predict(X_test_scaled)):.3f}")

# RBF Kernel (most popular for non-linear data)
svm_rbf = SVC(kernel='rbf', C=1.0, gamma='scale')
svm_rbf.fit(X_train_scaled, y_train)
print(f"RBF SVM Accuracy: {accuracy_score(y_test, svm_rbf.predict(X_test_scaled)):.3f}")

# Polynomial Kernel
svm_poly = SVC(kernel='poly', degree=3, C=1.0)
svm_poly.fit(X_train_scaled, y_train)
print(f"Polynomial (degree=3) Accuracy: {accuracy_score(y_test, svm_poly.predict(X_test_scaled)):.3f}")

# Support vectors count
print(f"Number of support vectors: {svm_rbf.n_support_}")
🟢 Output: Linear: 0.853 | RBF: 0.987 | Polynomial: 0.960 | Support vectors capture decision boundary

💡 Pro tip: Always scale features before SVM! SVM is sensitive to feature magnitudes. Use GridSearchCV to tune C and gamma for RBF kernel.

SVM: Advantages & Limitations

✅ Advantages

  • Effective in high-dimensional spaces
  • Memory efficient (uses only support vectors)
  • Versatile: different kernel functions for different data
  • Robust to overfitting (max margin principle)
  • Works well with clear margin separation

❌ Limitations

  • Not suitable for very large datasets (O(n²) to O(n³))
  • Less interpretable than decision trees
  • Sensitive to kernel and hyperparameter choice
  • Poor performance with overlapping classes
  • Requires feature scaling
