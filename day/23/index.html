What is Hierarchical Clustering?

Hierarchical Clustering builds a hierarchy of clusters by either merging (agglomerative) or splitting (divisive) clusters iteratively.

🌳 Key Features:
  • No need to specify number of clusters upfront
  • Produces a dendrogram for visualization
  • Deterministic (same results each time)
  • Can cut at any level to get desired clusters
📊 Real-World Applications:
  • Biology (phylogenetic trees)
  • Document clustering
  • Customer segmentation (with no fixed K)
  • Hierarchical taxonomies

📊 Hierarchical Structure

Root (All data)
Cluster A
Cluster B
A1
A2
B1
B2
Each level is a possible clustering

Agglomerative vs Divisive Clustering

Agglomerative (Bottom-Up)

Start with each point as its own cluster, then merge the closest clusters iteratively.

Step 1: Each point = cluster

Step 2: Merge closest clusters

Step 3: Repeat until 1 cluster remains

✅ Most common approach (sklearn implementation)

Divisive (Top-Down)

Start with all points in one cluster, then split recursively based on the farthest points.

Step 1: All points = 1 cluster

Step 2: Split cluster (farthest points)

Step 3: Repeat until each point = cluster

⚠️ Less common (computationally expensive)

📌 Scikit-learn uses Agglomerative Clustering (bottom-up approach) by default.

Linkage Methods: How to Measure Cluster Distance

📏 Distance Metrics:

Single Linkage:

Distance between closest points in clusters

d(A,B) = min(d(a,b)) for a∈A, b∈B

Complete Linkage:

Distance between farthest points in clusters

d(A,B) = max(d(a,b)) for a∈A, b∈B

Average Linkage:

Average distance between all points in clusters

d(A,B) = mean(d(a,b)) for a∈A, b∈B

Ward's Method:

Minimizes variance within clusters

Minimize sum of squared distances to centroid

📊 When to Use Which:

● Single Linkage: Elongated clusters, chain effect
● Complete Linkage: Compact, spherical clusters
● Average Linkage: Good general purpose
● Ward's Method: Most popular, minimizes variance

💡 Ward's is the default in scikit-learn for good reason!

Dendrogram: Visualizing Hierarchical Clustering

A Dendrogram is a tree diagram that shows the hierarchical clustering structure.

from scipy.cluster.hierarchy import dendrogram, linkage

linkage_matrix = linkage(X, method='ward')
dendrogram(linkage_matrix)
plt.show()

Reading a Dendrogram:

  • Height = distance between merged clusters
  • Cut at height h to get K clusters
  • Taller vertical lines = more dissimilar clusters

✂️ Cutting the Dendrogram

You can cut the dendrogram at different heights to get different numbers of clusters:

Cut High: Few clusters (coarse)
Cut Low: Many clusters (fine-grained)
Optimal Cut: Largest vertical gap

💡 In scikit-learn: Use n_clusters or distance_threshold to cut

Complete Implementation: Hierarchical Clustering

hierarchical_clustering_demo.py 
import numpy as np
import pandas as pd
from sklearn.datasets import make_blobs, make_moons, load_iris
from sklearn.cluster import AgglomerativeClustering
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import silhouette_score, adjusted_rand_score, adjusted_mutual_info_score
from scipy.cluster.hierarchy import dendrogram, linkage, fcluster
from scipy.spatial.distance import pdist
import matplotlib.pyplot as plt

# ========== 1. CREATE DATASET ==========
X, y_true = make_blobs(n_samples=300, centers=4, cluster_std=0.8, random_state=42)
print(f"Dataset: {X.shape[0]} samples, {X.shape[1]} features, {len(np.unique(y_true))} true clusters")

# ========== 2. SCALE DATA ==========
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# ========== 3. AGGLOMERATIVE CLUSTERING (SKLEARN) ==========
linkage_methods = ['ward', 'complete', 'average', 'single']
results = {}

for method in linkage_methods:
    try:
        model = AgglomerativeClustering(n_clusters=4, linkage=method)
        labels = model.fit_predict(X_scaled)
        silhouette = silhouette_score(X_scaled, labels)
        results[method] = {
            'labels': labels,
            'silhouette': silhouette,
            'n_clusters': len(np.unique(labels))
        }
        print(f"  {method}: Silhouette={silhouette:.4f}, Clusters={results[method]['n_clusters']}")
    except Exception as e:
        print(f"  {method}: Error - {e}")

# ========== 4. SCIKIT-LEARN DENDROGRAM ==========
linkage_matrix = linkage(X_scaled, method='ward')

print("\n📊 Linkage Matrix Shape:", linkage_matrix.shape)

# ========== 5. CUT DENDROGRAM AT DIFFERENT LEVELS ==========
# Cut at different distances to get different K
distances = [1, 2, 3, 4, 5, 8, 10]
print("\n📊 Cutting Dendrogram at different heights:")

for d in distances:
    labels_cut = fcluster(linkage_matrix, t=d, criterion='distance')
    n_clusters = len(np.unique(labels_cut))
    sil = silhouette_score(X_scaled, labels_cut)
    print(f"  Height={d}: {n_clusters} clusters, Silhouette={sil:.4f}")

# ========== 6. COMPARE WITH GROUND TRUTH ==========
best_method = max(results, key=lambda x: results[x]['silhouette'])
best_labels = results[best_method]['labels']

ari = adjusted_rand_score(y_true, best_labels)
ami = adjusted_mutual_info_score(y_true, best_labels)

print(f"\n📊 Best Method: {best_method}")
print(f"  Adjusted Rand Index (ARI): {ari:.4f}")
print(f"  Adjusted Mutual Info (AMI): {ami:.4f}")

# ========== 7. HIERARCHICAL CLUSTERING WITH DISTANCE THRESHOLD ==========
model_threshold = AgglomerativeClustering(
    n_clusters=None,
    distance_threshold=3.0,
    linkage='ward'
)
labels_threshold = model_threshold.fit_predict(X_scaled)
print(f"\n📊 Distance Threshold Clustering:")
print(f"  Clusters: {len(np.unique(labels_threshold))}")
print(f"  Silhouette: {silhouette_score(X_scaled, labels_threshold):.4f}")

# ========== 8. IRIS DATASET DEMO ==========
iris = load_iris()
X_iris, y_iris = iris.data, iris.target
X_iris_scaled = StandardScaler().fit_transform(X_iris)

model_iris = AgglomerativeClustering(n_clusters=3, linkage='ward')
labels_iris = model_iris.fit_predict(X_iris_scaled)

print(f"\n📊 Iris Dataset:")
print(f"  Silhouette: {silhouette_score(X_iris_scaled, labels_iris):.4f}")
print(f"  ARI vs true labels: {adjusted_rand_score(y_iris, labels_iris):.4f}")

# ========== 9. NON-LINEAR DATA (MOONS) ==========
X_moons, _ = make_moons(n_samples=200, noise=0.05, random_state=42)
X_moons_scaled = StandardScaler().fit_transform(X_moons)

model_moons = AgglomerativeClustering(n_clusters=2, linkage='single')
labels_moons = model_moons.fit_predict(X_moons_scaled)

print(f"\n📊 Moons Dataset:")
print(f"  Single Linkage Silhouette: {silhouette_score(X_moons_scaled, labels_moons):.4f}")

# Try complete linkage on moons
model_moons_complete = AgglomerativeClustering(n_clusters=2, linkage='complete')
labels_moons_complete = model_moons_complete.fit_predict(X_moons_scaled)
print(f"  Complete Linkage Silhouette: {silhouette_score(X_moons_scaled, labels_moons_complete):.4f}")

# ========== 10. SUMMARY ==========
print("\n📌 Hierarchical Clustering Key Takeaways:")
print("  ✅ Agglomerative is most common (bottom-up)")
print("  ✅ Ward's method is default (minimizes variance)")
print("  ✅ Dendrogram helps choose number of clusters")
print("  ✅ No need to specify K upfront")
print("  ❌ Computationally expensive for large datasets (O(n³))")
📊 Output: Ward's method best Silhouette: 0.67 | Dendrogram cut at height 3 gives 4 clusters | Hierarchical clustering is deterministic and interpretable!

💡 Key Insight: Hierarchical clustering is deterministic and doesn't require specifying K upfront. Use dendrograms to understand cluster structure. Ward's linkage is the most popular choice. For large datasets, use AgglomerativeClustering with a distance threshold.

Hierarchical Clustering vs K-Means

AspectHierarchical ClusteringK-Means
Number of ClustersNot required upfrontMust specify K
DeterministicYes (same results)No (random init)
InterpretabilityDendrogram visualizationCentroids only
ScalabilityO(n³) computationalO(n·K·d) fast
Cluster ShapeAny shape (with right linkage)Spherical only

Pros & Cons of Hierarchical Clustering

✅ Advantages

  • No need to specify number of clusters
  • Produces a dendrogram for visualization
  • Deterministic (reproducible results)
  • Works with any shape of clusters
  • Can use any distance metric

❌ Limitations

  • Computationally expensive O(n³)
  • Not suitable for large datasets
  • Results depend on linkage method
  • Cannot handle massive data
  • Determining optimal cut can be subjective
