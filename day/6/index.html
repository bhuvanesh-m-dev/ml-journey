Bootstrap Aggregation (Bagging)

What is Bootstrap?

Bootstrap is a statistical technique that creates multiple subsets of data by sampling with replacement from the original dataset.

📊 Original Dataset: [A, B, C, D, E]

📈 Bootstrap Sample 1: [A, B, B, C, E] (with replacement)

📈 Bootstrap Sample 2: [A, C, C, D, E]

📈 Bootstrap Sample 3: [B, B, D, E, E]

🎯 Bagging = Bootstrap + Aggregation. Train multiple models on bootstrapped samples and combine predictions.

Why Bagging Works

  • ✓ Reduces variance without increasing bias
  • ✓ Each model sees slightly different data → diverse perspectives
  • ✓ Parallel training → computationally efficient
  • ✓ Out-of-Bag (OOB) samples provide internal validation
For classification: Majority vote | For regression: Average prediction

Classification with Random Forest

🎯 How it works

Each tree in the forest "votes" for a class. The class with the majority of votes becomes the final prediction.

Example: Binary Classification (Spam/Not Spam)

Tree 1: Spam Tree 2: Spam Tree 3: Not Spam Tree 4: Spam

→ Final: Spam (3 vs 1)

📊 Key Metrics

  • • Out-of-Bag (OOB) Score: Internal validation using ~37% of data not in bootstrap
  • • Feature Importance: Measure how much each feature reduces impurity
  • • Confidence: Proportion of trees voting for predicted class

Regression with Random Forest

📈 How it works

Each tree predicts a continuous value. The forest averages all tree predictions for the final output.

Example: House Price Prediction ($)

Tree 1:$450k
Tree 2:$480k
Tree 3:$460k
Average:$463k

📊 Key Metrics

  • • R² Score: Coefficient of determination
  • • MSE/MAE: Mean squared/absolute error
  • • Prediction Intervals: Variance across trees gives uncertainty estimate

Why Random Forest Reduces Variance

\[ \text{Var}(\text{Average of N trees}) = \rho \sigma^2 + \frac{1 - \rho}{N} \sigma^2 \]

Where \(\rho\) = correlation between trees, \(\sigma^2\) = variance of individual trees

🔹 Low correlation (ρ small):

Random Forest reduces variance dramatically by decorrelating trees through feature sampling.

🔹 Many trees (N large):

Variance approaches ρσ², which is much smaller than individual tree variance σ².

Implementation: Random Forest

random_forest_demo.py 
from sklearn.ensemble import RandomForestClassifier, RandomForestRegressor
from sklearn.datasets import load_breast_cancer, fetch_california_housing
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score, mean_squared_error, r2_score

# ========== CLASSIFICATION ==========
data = load_breast_cancer()
X_train, X_test, y_train, y_test = train_test_split(data.data, data.target, random_state=42)

rf_clf = RandomForestClassifier(n_estimators=100, max_features='sqrt', random_state=42)
rf_clf.fit(X_train, y_train)
y_pred = rf_clf.predict(X_test)
print(f"Classification Accuracy: {accuracy_score(y_test, y_pred):.4f}")
print(f"Feature Importance: {rf_clf.feature_importances_[:5]}")

# ========== REGRESSION ==========
housing = fetch_california_housing()
X_train, X_test, y_train, y_test = train_test_split(housing.data, housing.target, random_state=42)

rf_reg = RandomForestRegressor(n_estimators=100, max_features='sqrt', random_state=42)
rf_reg.fit(X_train, y_train)
y_pred = rf_reg.predict(X_test)
print(f"Regression R² Score: {r2_score(y_test, y_pred):.4f}")
print(f"MSE: {mean_squared_error(y_test, y_pred):.4f}")
🌲 Output: Classification Accuracy: 0.9649 | Regression R²: 0.805 | OOB Score available with oob_score=True

💡 Key parameters to tune: n_estimators (number of trees), max_depth (tree depth), max_features (features per split), min_samples_split, bootstrap (whether to use bootstrapping).

Random Forest: Pros & Cons

✅ Advantages

  • High accuracy and robustness to overfitting
  • Handles both numerical and categorical features well
  • Provides feature importance scores
  • Handles missing data effectively
  • Parallelizable → scales well with multiple cores
  • No need for extensive hyperparameter tuning

❌ Limitations

  • Less interpretable than a single decision tree
  • Requires more memory and storage
  • Slower prediction compared to simpler models
  • May overfit with noisy regression tasks
  • Not ideal for very high-dimensional sparse data

# 🌳 Day 6: Random Forest — Ensemble of Decision Trees

  • ✅ Bootstrap Sampling: Create N datasets by sampling with replacement
  • ✅ Bagging = Bootstrap + Aggregation (reduce variance, combat overfitting)
  • ✅ Random Forest = Bagging + Random feature subset at each split
  • ✅ Classification: Majority vote across all trees
  • ✅ Regression: Average prediction across all trees
  • ✅ Feature randomization decorrelates trees → lower variance than bagging alone
