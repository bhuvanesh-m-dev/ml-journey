Bayes' Theorem: The Foundation

\[ P(A|B) = \frac{P(B|A) \cdot P(A)}{P(B)} \]

Where:

  • P(A|B) = Posterior Probability (What we want to find)
  • P(B|A) = Likelihood (Probability of evidence given hypothesis)
  • P(A) = Prior Probability (Initial belief)
  • P(B) = Evidence Probability (Normalizing constant)

Intuitive Explanation

Bayes' Theorem describes the probability of an event based on prior knowledge of conditions that might be related to the event.

🎯 Real-life analogy:

"The probability of having a disease given a positive test result depends on how accurate the test is AND how common the disease is in the population."

The "Naive" Assumption: Conditional Independence

Naive Bayes assumes that all features are independent given the class label. This is rarely true in reality, but the algorithm still works surprisingly well!

\[ P(C|X_1, X_2, ..., X_n) \propto P(C) \prod_{i=1}^{n} P(X_i|C) \]

Features are conditionally independent given class C

🎯 Why "Naive"?

  • • In reality, features like "age" and "income" ARE correlated
  • • But the naive assumption simplifies calculations drastically
  • • Works well for text classification, spam filtering, sentiment analysis

Types of Naive Bayes Classifiers

Gaussian NB

For continuous features assumed to follow a normal distribution.

P(x_i|C) = (1/√(2πσ²))·exp(-(x-μ)²/2σ²)
📊 Example: Height, Weight, Temperature

Multinomial NB

For discrete features like word counts (text classification).

P(x_i|C) = (count of word i in class C + α) / (total words in C + α·n)
📄 Example: Spam detection, document classification

Bernoulli NB

For binary features (presence/absence).

P(x_i|C) = p^y · (1-p)^(1-y)
🎯 Example: Word presence (not count), binary indicators

Worked Example: Spam Classification

Suppose we want to classify an email containing the word "free" as Spam or Not Spam.

Training Data:

• 100 total emails: 30 Spam, 70 Not Spam

• "free" appears in: 20 Spam emails, 5 Not Spam emails

Probabilities:

P(Spam) = 30/100 = 0.3

P(Not Spam) = 70/100 = 0.7

P("free" | Spam) = 20/30 = 0.667

P("free" | Not Spam) = 5/70 = 0.071

Applying Bayes' Theorem:

P(Spam | "free") ∝ P(Spam) × P("free" | Spam) = 0.3 × 0.667 = 0.200

P(Not Spam | "free") ∝ P(Not Spam) × P("free" | Not Spam) = 0.7 × 0.071 = 0.050

After normalization: P(Spam | "free") = 0.200 / (0.200 + 0.050) = 0.800 (80%)

P(Not Spam | "free") = 0.050 / 0.250 = 0.200 (20%)

🎯 Prediction: SPAM (80% probability)

🧴 Laplace Smoothing (Add-α Smoothing)

When a feature never appears in training for a class, P(feature|class) = 0 → whole probability becomes 0. Laplace smoothing fixes this:

\[ P(x_i|C) = \frac{\text{count}(x_i, C) + \alpha}{\text{total}(C) + \alpha \cdot n} \]

α = 1 (Laplace), α = 0.5 (Lidstone). Prevents zero probabilities.

Implementation: Scikit-learn Naive Bayes

naive_bayes_demo.py 
from sklearn.naive_bayes import GaussianNB, MultinomialNB, BernoulliNB
from sklearn.feature_extraction.text import CountVectorizer
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score, classification_report
from sklearn.datasets import load_iris, fetch_20newsgroups

# ========== GAUSSIAN NB (Continuous Features) ==========
iris = load_iris()
X_train, X_test, y_train, y_test = train_test_split(iris.data, iris.target, random_state=42)

gnb = GaussianNB()
gnb.fit(X_train, y_train)
y_pred = gnb.predict(X_test)
print(f"Gaussian NB Accuracy: {accuracy_score(y_test, y_pred):.3f}")
print(f"Class Priors: {gnb.class_prior_}")

# ========== MULTINOMIAL NB (Text Classification) ==========
categories = ['alt.atheism', 'soc.religion.christian']
newsgroups = fetch_20newsgroups(subset='all', categories=categories, shuffle=True, random_state=42)

vectorizer = CountVectorizer(stop_words='english', max_features=5000)
X = vectorizer.fit_transform(newsgroups.data)
y = newsgroups.target

X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=42)

mnb = MultinomialNB(alpha=1.0)  # Laplace smoothing
mnb.fit(X_train, y_train)
y_pred = mnb.predict(X_test)
print(f"Multinomial NB Accuracy: {accuracy_score(y_test, y_pred):.3f}")
print(classification_report(y_test, y_pred, target_names=categories))
📊 Output: Gaussian NB: 0.973 | Multinomial NB: 0.967 | Fast, probabilistic, great for text!

💡 When to use Naive Bayes: Text classification (spam detection, sentiment analysis), high-dimensional data, real-time prediction (very fast), small datasets, multi-class problems.

Naive Bayes: Strengths & Weaknesses

✅ Advantages

  • Extremely fast training and prediction
  • Works well with high-dimensional data (text, genomics)
  • Handles missing data naturally
  • Provides probabilistic predictions
  • Requires less training data than complex models
  • Highly scalable (O(nd) training time)

❌ Limitations

  • Naive independence assumption rarely true in real data
  • Poor probability estimates (calibration issues)
  • Cannot learn feature interactions
  • Zero frequency problem (requires smoothing)
  • Less accurate than ensemble methods for complex tasks

🌍 Real-World Applications

📧 Spam Filtering - Gmail, Outlook
❤️ Sentiment Analysis - Product reviews, Twitter
📄 Document Classification - News categorization
🩺 Medical Diagnosis - Disease prediction
