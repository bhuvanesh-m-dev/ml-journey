What is a Decision Tree?

A Decision Tree is a non-parametric supervised learning algorithm used for classification and regression. It models decisions as a tree structure:

  • Root Node: Topmost node, represents the entire dataset.
  • Decision Nodes: Internal nodes testing a feature.
  • Leaf Nodes: Final prediction (class label or value).

Entropy & Information Gain

📊 Entropy

Measures impurity/uncertainty in a dataset. Higher entropy → more disorder.

\[ H(S) = -\sum_{i=1}^{c} p_i \log_2(p_i) \]

Where \(p_i\) = proportion of class i in dataset S, c = number of classes.

Pure node: H=0 Max impurity (50/50): H=1

📈 Information Gain

Reduction in entropy after splitting on a feature. Higher IG → better split.

\[ IG(S, A) = H(S) - \sum_{v \in Values(A)} \frac{|S_v|}{|S|} H(S_v) \]

We choose feature with maximum Information Gain at each split.

▶ Example: Dataset with 9 Yes, 5 No → H = -[(9/14)log₂(9/14) + (5/14)log₂(5/14)] = 0.94 bits. After splitting on feature, weighted entropy reduces.

Gini Impurity (CART Algorithm)

\[ Gini(S) = 1 - \sum_{i=1}^{c} p_i^2 \]

Gini measures how often a randomly chosen element would be misclassified. Lower Gini = purer node. Used by CART (Classification and Regression Trees).

Pure node → Gini = 0
Equal classes → Gini = 0.5

Comparison: Entropy vs Gini

  • Both measure impurity, but Gini is computationally faster.
  • Entropy tends to produce more balanced trees, Gini favors majority class.
  • In practice, performance difference is minimal.

Scikit-learn default: Gini, but you can set criterion="entropy".

Avoiding Overfitting: Pruning

Overfitting

Tree grows too deep, memorizes noise in training data → poor generalization.

Pruning Techniques

  • Pre-pruning: Stop early (max_depth, min_samples_split).
  • Post-pruning: Grow full tree, then remove branches with little importance.
  • Cost-complexity: Use ccp_alpha to penalize complexity.

Hyperparameters

max_depth: 3-10
min_samples_split: 2-20
min_samples_leaf: 1-10
max_features: 'sqrt', 'log2'

Implementation with Scikit-learn

decision_tree_demo.py 
from sklearn.tree import DecisionTreeClassifier
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score, confusion_matrix
from sklearn.tree import plot_tree
import matplotlib.pyplot as plt

# Load Iris dataset
iris = load_iris()
X_train, X_test, y_train, y_test = train_test_split(iris.data, iris.target, random_state=42)

# Train Decision Tree (Entropy criterion)
clf = DecisionTreeClassifier(criterion='entropy', max_depth=3, random_state=42)
clf.fit(X_train, y_train)

# Evaluate
y_pred = clf.predict(X_test)
print(f"Accuracy: {accuracy_score(y_test, y_pred):.3f}")
print(f"Feature importance: {clf.feature_importances_}")

# Visualize tree (optional)
plt.figure(figsize=(12,8))
plot_tree(clf, filled=True, feature_names=iris.feature_names, class_names=iris.target_names)
plt.show()
🌳 Output: Accuracy = 0.973, Feature importance reveals most predictive features.

💡 Key insight: Decision trees are interpretable — you can trace any prediction along branches. Feature importance shows which attributes drive decisions.

Pros & Cons

Advantages

  • Easy to understand and interpret (white-box model).
  • Requires little data preprocessing (no scaling needed).
  • Handles both numerical and categorical data.
  • Non-parametric → no assumption about data distribution.

Disadvantages

  • Prone to overfitting (requires pruning/ensembles).
  • Can be unstable: small data changes lead to different trees.
  • Biased towards features with many levels.
  • Solution: Random Forest / Gradient Boosting (ensemble methods).

# 🌳 Day 3: Decision Trees — Key Takeaways

  • ✅ Decision trees split nodes based on Entropy / Information Gain or Gini Impurity.
  • ✅ ID3 algorithm uses Information Gain; CART uses Gini Index.
  • ✅ Recursive binary splitting: choose feature that minimizes impurity at each step.
  • ✅ Overfitting controlled via pre-pruning (max_depth) or post-pruning.
  • ✅ Decision trees form the backbone of ensemble methods (Random Forest, XGBoost).
