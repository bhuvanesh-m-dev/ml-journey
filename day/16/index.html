Why Encode Categorical Data?

🤖 Machine Learning Requirement

Most ML algorithms only work with numerical data. Categorical text must be converted to numbers.

📊 Mathematical Operations

Distance calculations, gradient descent, and matrix operations require numeric inputs.

⚠️ Ordinal vs Nominal

Different encoding methods preserve different relationships in the data.

Label Encoding: Converting Categories to Numbers

Label Encoding assigns a unique integer to each category (0, 1, 2, ...). Best for ordinal data where order matters.

# Example: Education Levels
import pandas as pd
from sklearn.preprocessing import LabelEncoder

df = pd.DataFrame({'education': ['High School', 'Bachelor', 'Master', 'PhD', 'Bachelor']})

le = LabelEncoder()
df['education_encoded'] = le.fit_transform(df['education'])

# Result:
# High School → 0
# Bachelor → 1
# Master → 2
# PhD → 3

✅ Best for: Ordinal categories (Small, Medium, Large), Education levels, Rating scales (1-5)

⚠️ Important Warning:

Label Encoding implies ordinal relationships (0 < 1 < 2). This can mislead models like linear regression or SVM if used on nominal data (colors, countries).

❌ Bad example: Red=0, Blue=1, Green=2 implies Red < Blue < Green (nonsense!)

✅ Good example: Cold=0, Warm=1, Hot=2 (order exists)

One-Hot Encoding: Binary Columns for Each Category

One-Hot Encoding creates a new binary column for each unique category. Best for nominal data with no order.

import pandas as pd
from sklearn.preprocessing import OneHotEncoder

df = pd.DataFrame({'color': ['Red', 'Blue', 'Green', 'Blue', 'Red']})

# Using pandas get_dummies
onehot_df = pd.get_dummies(df, prefix='color')

# Result:
# color_Red  color_Blue  color_Green
#     1          0           0
#     0          1           0
#     0          0           1
#     0          1           0
#     1          0           0

✅ Advantages & Disadvantages:

✓ No ordinal assumption — safe for nominal data
✓ Works with linear models and distance-based algorithms
✗ Creates many columns (curse of dimensionality)
✗ Can cause multicollinearity (use drop='first')

import pandas as pd
import numpy as np
from sklearn.preprocessing import LabelEncoder, OneHotEncoder
from sklearn.compose import ColumnTransformer
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

# ========== CREATE SAMPLE DATASET ==========
df = pd.DataFrame({
    'size': ['Small', 'Medium', 'Large', 'Medium', 'Small', 'Large'],
    'color': ['Red', 'Blue', 'Green', 'Blue', 'Red', 'Green'],
    'city': ['NYC', 'LA', 'Chicago', 'NYC', 'LA', 'Chicago'],
    'target': [1, 0, 1, 0, 1, 0]
})

print("Original DataFrame:")
print(df)

# ========== METHOD 1: LABEL ENCODING (for ordinal 'size') ==========
# Define ordinal mapping
size_order = {'Small': 0, 'Medium': 1, 'Large': 2}
df['size_label'] = df['size'].map(size_order)

# Using sklearn LabelEncoder (automatic mapping)
le = LabelEncoder()
df['city_label'] = le.fit_transform(df['city'])
print("\nAfter Label Encoding:")
print(df[['size', 'size_label', 'city', 'city_label']])

# ========== METHOD 2: ONE-HOT ENCODING (for nominal 'color') ==========
# Using pandas get_dummies (easiest)
onehot_df = pd.get_dummies(df, columns=['color'], prefix='color')
print("\nAfter One-Hot Encoding (pandas):")
print(onehot_df[['color_Blue', 'color_Green', 'color_Red']])

# Using sklearn OneHotEncoder
encoder = OneHotEncoder(sparse_output=False, drop='first')  # drop first to avoid multicollinearity
color_encoded = encoder.fit_transform(df[['color']])
color_df = pd.DataFrame(color_encoded, columns=encoder.get_feature_names_out(['color']))

# ========== METHOD 3: ColumnTransformer (Combine encodings) ==========
X = df.drop('target', axis=1)
y = df['target']

preprocessor = ColumnTransformer([
    ('size_ordinal', LabelEncoder(), ['size']),  # Note: LabelEncoder expects 1D
    ('color_onehot', OneHotEncoder(drop='first'), ['color']),
    ('city_onehot', OneHotEncoder(drop='first'), ['city'])
])

# ========== METHOD 4: Handle Unseen Categories ==========
encoder_robust = OneHotEncoder(handle_unknown='ignore', sparse_output=False)

# ========== FULL ML PIPELINE EXAMPLE ==========
# Create larger dataset for ML demo
np.random.seed(42)
n_samples = 500
df_large = pd.DataFrame({
    'size': np.random.choice(['Small', 'Medium', 'Large'], n_samples),
    'color': np.random.choice(['Red', 'Blue', 'Green'], n_samples),
    'value': np.random.randn(n_samples)
})

# Ordinal mapping for size
size_map = {'Small': 0, 'Medium': 1, 'Large': 2}
df_large['size_encoded'] = df_large['size'].map(size_map)

# One-hot encode color
df_large = pd.get_dummies(df_large, columns=['color'], prefix='color')

# Create target based on encoded features
df_large['target'] = (df_large['size_encoded'] * 0.5 + 
                     df_large['color_Blue'] * 0.3 + 
                     df_large['color_Green'] * 0.2 + 
                     df_large['value'] * 0.1 > 0.8).astype(int)

X_ml = df_large.drop('target', axis=1)
y_ml = df_large['target']

X_train, X_test, y_train, y_test = train_test_split(X_ml, y_ml, test_size=0.2, random_state=42)

model = LogisticRegression(max_iter=1000)
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
print(f"\nModel Accuracy with encoded features: {accuracy_score(y_test, y_pred):.4f}")

print("\n🎯 Key Takeaways:")
print("   - Label Encoding: Use for ordinal data (size, rank)")
print("   - One-Hot Encoding: Use for nominal data (color, city)")
print("   - Always drop first column in OHE to avoid multicollinearity")
print("   - For high-cardinality features, consider frequency or target encoding")
📊 Output: Label encoding for ordinal 'size', One-hot for nominal 'color' | Model accuracy: 0.85 | Proper encoding prevents false ordinal assumptions!

💡 Best Practice: Always use Label Encoding for ordinal data and One-Hot Encoding for nominal data. For tree-based models (Random Forest, XGBoost), Label Encoding often works fine even for nominal data, but for linear models, One-Hot is safer!

📋 Decision Flow: Which Encoding to Choose?

❓ Question 1: Does the category have a natural order? (Small < Medium < Large)
→ YES: Label Encoding
→ NO: One-Hot Encoding (or other nominal methods)
❓ Question 2: How many unique categories?
→ < 15 categories: One-Hot works well
→ > 50 categories: Consider Frequency or Target Encoding

# 🔢 Day 16: Categorical Data Encoding — Label & One-Hot

  • ✅ Label Encoding — Assigns integers (0,1,2...) to categories. Best for ordinal data.
  • ✅ One-Hot Encoding — Creates binary columns for each category. Best for nominal data.
  • ✅ Key difference: Label implies order; One-Hot treats categories equally.
  • ✅ When to use Label: Size (S/M/L), Rating (1-5), Education level.
  • ✅ When to use One-Hot: Color, City, Gender, Product ID.
  • ✅ Implementation: pandas.get_dummies(), sklearn.LabelEncoder, sklearn.OneHotEncoder

📌 Pro Tip: For high-cardinality features, use Frequency Encoding or Target Encoding to avoid dimension explosion!
