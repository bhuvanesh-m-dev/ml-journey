Performance Optimization: Speed Up Your Code

Pandas operations can be optimized using vectorization, query(), eval(), and avoiding loops.

🚀 Vectorization vs Loops: 
# SLOW: Python loop
for i in range(len(df)):
    df.loc[i, 'new'] = df.loc[i, 'a'] * 2

# FAST: Vectorized
df['new'] = df['a'] * 2
📊 .query() for Filtering: 
# Cleaner & faster
df.query('age > 30 and salary > 50000')
⚡ .eval() for Arithmetic: 
df.eval('salary_adj = salary * 1.1 - bonus')

📊 Speed Comparison

Python Loop ~100 ms
.apply() ~30 ms
Vectorized ~2 ms
.eval() ~1 ms

Vectorization is 50-100x faster than loops!

Memory Optimization: Handling Large Data

📉 Downcasting Numeric Types: 
# Convert int64 to int8/16/32
for col in df.select_dtypes('int'):
    df[col] = pd.to_numeric(df[col], downcast='integer')

# Convert float64 to float32
for col in df.select_dtypes('float'):
    df[col] = pd.to_numeric(df[col], downcast='float')
📊 Categorical Data: 
# Convert object to category
df['city'] = df['city'].astype('category')
# 98% memory reduction!
📖 Reading Efficiently: 
df = pd.read_csv('large.csv', 
                 usecols=['col1','col2'],
                 dtype={'col1': 'int32'})

📊 Memory Savings Examples

int64 → int8 87.5% ↓
float64 → float32 50% ↓
object → category (10M rows, 5 unique) 98% ↓
string columns → .str operations ~40% ↓

💡 Use df.info(memory_usage='deep') to check memory usage

Parallel Processing: Multi-Core Magic

Use Dask, Modin, or Swifter to parallelize pandas operations across CPU cores.

⚡ Swifter (Easy): 
import swifter
df['new'] = df['col'].swifter.apply(lambda x: x**2)
🚀 Modin (Ray/Dask backend): 
import modin.pandas as mpd
df = mpd.read_csv('large.csv')
result = df.groupby('col').sum()
🌐 Dask (Distributed): 
import dask.dataframe as dd
df = dd.read_csv('large*.csv')
result = df.groupby('col').sum().compute()

📊 Speed Comparison (10M rows)

Pandas (single core) 120s
Swifter (parallel apply) 25s
Modin (Ray) 8s
Dask (8 cores) 4s

💡 Parallel processing is essential for large datasets > 10GB!

Handling Data That Doesn't Fit in Memory

📂 Chunking: 
chunk_size = 100000
chunks = []
for chunk in pd.read_csv('huge.csv', chunksize=chunk_size):
    processed = chunk.groupby('col').sum()
    chunks.append(processed)
result = pd.concat(chunks).groupby('col').sum()
💾 Efficient File Formats: 
# Parquet (columnar, compressed)
df.to_parquet('data.parquet')
df = pd.read_parquet('data.parquet')

# Feather (fast read/write)
df.to_feather('data.feather')
df = pd.read_feather('data.feather')

📊 File Format Comparison

CSV (text, uncompressed) Slow, large
Parquet (binary, compressed) 5-10x faster, 80% smaller
Feather (binary, uncompressed) 100x faster read/write
HDF5 (binary, compressed) Good for complex data

💡 Parquet is the modern standard for big data!

Building Production-Ready Pipelines

from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.ensemble import RandomForestClassifier

# Define preprocessing pipeline
preprocessor = ColumnTransformer([
    ('num', StandardScaler(), ['age', 'income']),
    ('cat', OneHotEncoder(), ['city', 'department'])
])

# Full ML pipeline
pipeline = Pipeline([
    ('preprocessor', preprocessor),
    ('classifier', RandomForestClassifier())
])

# Fit and predict
pipeline.fit(X_train, y_train)
predictions = pipeline.predict(X_test)

# Save pipeline
import joblib
joblib.dump(pipeline, 'model_pipeline.pkl')

📦 Pipeline Best Practices

  • Use ColumnTransformer for mixed data types
  • Use Pipeline to chain preprocessing + model
  • Save pipelines with joblib or pickle
  • Use FunctionTransformer for custom functions
  • Add validation steps (check for nulls, types)
  • Log parameters and metrics for monitoring

💡 Production pipelines ensure reproducibility and easy deployment!

    'income': np.random.randint(30000, 200000, n),

    'department': np.random.choice(['Sales', 'Engineering', 'HR'], n),

})

print("\n📊 Memory Optimization:")

# Check dtypes

print(df.dtypes.value_counts())

# Downcast integers
for col in df.select_dtypes(include=['int64']).columns:

# Convert object to category
for col in df.select_dtypes(include=['object']).columns:
    df[col] = df[col].astype('category')

print("After optimization:")
print(df.dtypes.value_counts())
print(f"New memory: {df.memory_usage(deep=True).sum() / 1024**2:.2f} MB")

# ========== 3. PERFORMANCE COMPARISON ==========
print("\n📊 Performance Comparison (10k rows):")
