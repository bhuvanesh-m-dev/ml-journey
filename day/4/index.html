What is K-Nearest Neighbors?

KNN is a non-parametric, lazy learning algorithm used for classification and regression. It predicts based on the majority vote (classification) or average (regression) of its K nearest neighbors.

📌 Key principle: "Birds of a feather flock together" — similar data points exist in close proximity.

✅ No explicit training phase ✅ Instance-based learning ✅ Stores all training data

Euclidean Distance Formula

\[ d(p, q) = \sqrt{\sum_{i=1}^{n} (q_i - p_i)^2} \]

where p and q are points in n-dimensional space

Euclidean distance measures the straight-line distance between two points. In KNN, we compute distance from the query point to every training point, then select the K closest.

📐 2D Example:

Point A: (x₁, y₁) = (2, 3) Point B: (x₂, y₂) = (5, 7)
\[ d = \sqrt{(5-2)^2 + (7-3)^2} = \sqrt{3^2 + 4^2} = \sqrt{9 + 16} = \sqrt{25} = 5 \]

✅ Distance = 5 units

📐 Important: KNN uses distance metrics to find nearest neighbors. Euclidean is most common, but other metrics exist (Manhattan, Minkowski, Cosine).

Choosing the Optimal K

Small K (e.g., K=1)

High variance, sensitive to noise, may overfit. Decision boundary is very irregular.

⚠️ Prone to outliers

Optimal K (Cross-validated)

Balances bias-variance tradeoff. Usually K = √N (where N = sample size) works as heuristic.

✅ Best generalization

Large K

High bias, oversmooths decision boundaries, underfits. May include too many distant points.

⚠️ Loss of local patterns

💡 Rule of thumb: K should be odd for binary classification to avoid ties. Use cross-validation (e.g., GridSearchCV) to find optimal K.

Implementation: KNN Classifier

knn_from_scratch.py 
import numpy as np
from sklearn.neighbors import KNeighborsClassifier
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score

# Euclidean Distance Function
def euclidean_distance(a, b):
    return np.sqrt(np.sum((a - b) ** 2))

# KNN from scratch (simplified)
class KNN_Scratch:
    def fit(self, X, y):
        self.X_train = X
        self.y_train = y
    
    def predict(self, X, k=3):
        predictions = []
        for test_point in X:
            distances = [euclidean_distance(test_point, train_point) for train_point in self.X_train]
            k_indices = np.argsort(distances)[:k]
            k_labels = [self.y_train[i] for i in k_indices]
            predictions.append(max(set(k_labels), key=k_labels.count))
        return np.array(predictions)

# Load dataset & test
iris = load_iris()
X_train, X_test, y_train, y_test = train_test_split(iris.data, iris.target, random_state=42)

# Using Scikit-learn KNN
knn_sk = KNeighborsClassifier(n_neighbors=5, metric='euclidean')
knn_sk.fit(X_train, y_train)
y_pred_sk = knn_sk.predict(X_test)
print(f"Scikit-learn KNN Accuracy: {accuracy_score(y_test, y_pred_sk):.3f}")
🎯 Output: Scikit-learn KNN Accuracy: 0.973 | Euclidean distance used by default

💡 Key insight: KNN is a lazy learner — all computation happens at prediction time. The Euclidean distance calculation is the core operation. Normalizing features (scaling) is essential because distance metrics are sensitive to feature scales.

⚠️ Feature Scaling is CRITICAL for KNN

Since KNN relies on distance calculations, features with larger ranges dominate the metric. Always apply StandardScaler or MinMaxScaler before training KNN.

KNN: Advantages & Limitations

✅ Advantages

  • Simple, intuitive, and easy to implement.
  • No training phase → fast training (lazy learning).
  • Non-parametric → no assumptions about data distribution.
  • Works well with multi-class problems.
  • Naturally handles non-linear decision boundaries.

❌ Limitations

  • Computationally expensive at prediction time (O(N) per query).
  • Memory-intensive: stores all training data.
  • Sensitive to irrelevant features and feature scaling.
  • Curse of dimensionality: performance degrades in high dimensions.
  • Choosing optimal K requires cross-validation.
